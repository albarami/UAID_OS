## 0. Scope & non-goals
- **Scope.** A **deterministic-gated, LLM-assisted, DESCRIPTIVE-ONLY** spine-level semantic contradiction detector over the project's `requirement`+`acceptance_criterion` artifacts (title/body via `list_artifacts`). It detects **pairwise** semantic contradictions, classifies each by the **§6.4 8 types**, pins both conflicting artifacts via **FK-proven** provenance, and records immutable, append-only **decision requests** across two tables. LLM-assisted (FakeLLM in CI) reusing the Slice‑35/36 pipeline. **SEPARATE** from Slice‑13.

Stored `description` may carry source-derived material — **no denylist/redaction/no-secret guarantee**; audit/logs never carry it. FakeLLM only.

## 3. Pure module — `app/intake/semantic_contradictions.py`
### 3.1 constants + taxonomy
`CONFLICT_TYPES = (minor_wording, scope, business_rule, technical, legal_regulatory, security, budget_timeline, authority)` — **exactly the §6.4 8** (B3, no `unclassified`). `OUTCOMES = (succeeded, skipped_insufficient_input, refused_injection, blocked_by_budget, failed)`. `RULESET_VERSION="slice37.v1"`; `PROMPT_VERSION="semantic_contradiction.v1"`. Bounds: `MAX_DESCRIPTION_CHARS=2000`, `MAX_ANALYZED_ARTIFACTS=200`, `MAX_ARTIFACT_BODY_CHARS_IN_PROMPT=4000`, `MAX_CONTRADICTIONS_PERSISTED=200`.
### 3.2 prompt + format
`DETECT_SYSTEM_PROMPT` (untrusted-data framed; "**Do not resolve or choose a side**; only detect PAIRWISE contradictions, classify each as one of the 8 types, and cite the two conflicting item refs **verbatim** from the provided set"; STRICT JSON `{contradictions:[{conflict_type, item_a, item_b, description}]}` — `item_a`/`item_b` are the **opaque per-prompt item keys** shown, never bare refs — B8). `format_artifacts(artifacts) -> (block, key_to_artifact)`: assigns each artifact a unique sequential **item key** `A1`,`A2`,… (1:1 to the artifact for THIS prompt — B8, since a bare `ref` is not unique across kinds), renders a `[A1] (kind ref) title / body[:MAX_ARTIFACT_BODY_CHARS_IN_PROMPT]` block, and returns the block + the `key_to_artifact` map; sets `input_truncated` if any body or the count was capped.
### 3.3 parse + validate (fail-closed)
`parse_contradictions(raw) -> list[ContradictionDraft]` (strict JSON; `ContradictionDraft(conflict_type, item_a, item_b, description)` — `item_*` are item keys, not refs; malformed ⇒ `SemanticContradictionParseError`). `keep_valid(drafts, key_to_artifact: dict) -> list[…]`: **DROP** any draft whose `conflict_type ∉ CONFLICT_TYPES` (B3), or `item_a == item_b`, or either item key ∉ `key_to_artifact` (B4/B8 — each key resolves 1:1 to a single artifact; ambiguous/unknown ⇒ dropped fail-closed); resolve `item_a`/`item_b` → `artifact_a_id`/`artifact_b_id` (FK-backed); truncate `description` to `MAX_DESCRIPTION_CHARS`; cap the surviving list at `MAX_CONTRADICTIONS_PERSISTED` (`input_truncated=true` if capped). Cost rule = incurred-cost (Slice‑35, B2).
