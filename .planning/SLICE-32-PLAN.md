## 0. Goal & non-goal
- **Goal.** A deterministic, tenant-owned **secret-reference verification store** + a **broker-gated** verifier that confirms each declared `secrets_and_credentials_manifest` reference **resolves in its approved manager** — producing "secrets available" evidence (R5 App. A `:2968`; §26.3 `:2471`) — **without ever storing, logging, auditing, or returning a secret value** (template-17 `note`; spec:1094 *verify exists*). The store records only `(manager, reference_name, outcome, resolved-boolean)`.
- **Non-goal — NO A5 gate flip / NO readiness change this slice** (roadmap `:265`: "no gate flips alone"). `production_autonomy.py` and `readiness.py` are **UNTOUCHED**; `ruleset_version` stays `slice31.v1`. The store is a *future* feed for gate #2 (deploy readiness) + R5 completeness; a later slice consumes it. **No secret rotation/mutation** (that's the mandatory-approval `change_secrets`, out of scope). **No secret value is stored, logged, audited, persisted, returned, or bound into any DB/broker/audit payload** — the only allowed value contact is transient in-process `env` inspection to compute non-emptiness (§1). No HTTP endpoint. No real manager network in tests.

## 1. The invariant that dominates this slice — ZERO secret-value leakage (B4 — precise)
A secret **value** must never be **stored, logged, audited, persisted, returned, or bound into any DB / broker-param / audit payload**. The `env` connector **may transiently inspect the value in process memory only enough to compute non-emptiness** (the boolean) — it must not assign it to anything that is stored/logged/returned. Defenses, layered:
- **Structural:** the schema has **no value column** — there is nowhere to put one.
- **Connector contract:** the verifier returns only an `outcome` enum + `resolved` boolean. For `env`, it computes `bool((os.environ.get(name) or "").strip())` and discards the value immediately — the value is never returned, logged, or stored.
- **Input denylist (defense-in-depth):** the value-key denylist (`value`/`secret`/`password`/`token`/… as **dict keys**) is already enforced upstream at category-declare time (`categories.py:74-80,127`); the verifier consumes the already-clean persisted shape and does not re-introduce values. (This denylist is **never** applied as a substring filter on the `reference_name` string — B2.)
- **Audit/params:** safe metadata only — `manager` + `outcome` + counts; **never** `reference_name` (defense-in-depth — a name is not a value, but is kept out of broker params + audit anyway) and **never** a value.
- A dedicated test sets a real env var to a **sentinel value** and asserts the sentinel reaches **no** snapshot row / audit payload / `tool_calls.params` / log.

## 2. Bound decisions (D-32-1…9)
- **D-32-1 Managers — BOUND (B1):** `SUPPORTED_MANAGERS=("env",)` is the **verifiable** set this slice (matches the `has_declared_credential` precedent; **local** check, so **no network / no SSRF surface**). The `manager` column persists the **original declared identifier** as **bounded safe text** (shape `^[a-z0-9_.:-]{1,64}$`), NOT a `('env')` enum — so a `vault`/cloud reference can be recorded. **DB-enforced rule:** `manager <> 'env'  ⟹  outcome='unsupported_manager' AND resolved=false`. Non-`env` managers are otherwise **deferred** (Vault/AWS/GCP, follow-up).
- **D-32-2 Honesty model — BOUND** (Slice-31 B4 lesson): `outcome ∈ {resolved, not_found, unsupported_manager, probe_error}`; `resolved = (outcome == 'resolved')` (DB-enforced). A check that could not be performed is **never** `not_found`.
- **D-32-2b `reference_name` validation — BOUND (B2):** a **bounded safe shape only** — `^[A-Za-z0-9_./:-]{1,256}$` — which **accepts legitimate names** like `prod/db_password` and `app/api_key` (per `tests/test_intake_categories.py:91-100`). **No** token-prefix / value-substring denylist is applied to the name string (that would wrongly reject real names). The value-key denylist is a payload-**key** check, enforced upstream by `categories.py`.
- **D-32-3 What "resolve" means for `env` — BOUND (B4):** the env var named `reference_name` is **set AND non-empty after strip** ⇒ `resolved`; unset/empty ⇒ `not_found`. The connector inspects the value transiently in memory **only** to compute the non-empty boolean, then discards it — it is never stored/logged/returned/audited (§1).
- **D-32-4 Store granularity — BOUND:** one immutable row per declared reference per check; **latest-wins per `(manager, reference_name)`** (mirrors the Slice-26/28/30/31 latest-wins pattern).
- **D-32-5 Broker tool/action — BOUND (B3):** new tool `secrets.verify_reference` → **new read action `verify_secret_reference`** (`_r(L.A1)`, **non-mandatory, no approval**) — distinct from the existing mutating `change_secrets` (A1 + mandatory-approval, untouched). **Safe params `{manager, reference_present: true}`** — the key is `reference_present`, NOT `secret_present` (which `sanitize_params` would redact via the "secret" marker, `registry.py:15-23`). §16.4 broker chokepoint applies.
- **D-32-6 `reference_name` handling — BOUND:** persisted in the snapshot (it is a *name*, already openly declared in `intake_categories`), but kept **out of** broker params + audit (defense-in-depth). Never any value.
- **D-32-7 No-gate-flip — BOUND:** `production_autonomy.py` + `readiness.py` untouched; ruleset stays `slice31.v1`; a `before == after` production-autonomy regression test proves it.
- **D-32-8 Provenance — BOUND:** two-tier (`caller_supplied_unverified` | `connector_verified`); the connector path stamps `connector_verified`; a caller path may write `caller_supplied_unverified` only (cannot assert `connector_verified`).
- **D-32-9 Trigger — BOUND:** admin/internal service method; **no HTTP endpoint** this slice.
- **D-32-10 Declared-reference extraction — BOUND (B5):** the verifier consumes **only the canonical persisted `intake_categories` shape** for `secrets_and_credentials_manifest` — top-level `{manager, reference_name}` **or** `{references: [{manager, reference_name}, …]}` (`categories.py:153-164`; the `has_declared_credential` read pattern). **Template-YAML normalization** (file 17's `secret_manager` / `secrets[]`) is **explicitly out of scope** this slice. Malformed/missing references fail closed (no row written for them).

## 3. Pure module — `app/release/secrets_verification.py`
- `SUPPORTED_MANAGERS = ("env",)` (verifiable set); `MANAGER_RE = ^[a-z0-9_.:-]{1,64}$` (bounded safe text for the *persisted* manager identifier — any declared manager, B1); `OUTCOMES = ("resolved","not_found","unsupported_manager","probe_error")`; `PROVENANCES`/`WRITABLE`/`CONNECTOR_WRITABLE` (two-tier).
- `REFERENCE_NAME_RE = ^[A-Za-z0-9_./:-]{1,256}$` — bounded safe **shape only**; **accepts** `prod/db_password`, `app/api_key` (B2). No token/value-substring rejection on the name.
- `validate_new_secret_check` / `validate_connector_secret_check`: `manager` matches `MANAGER_RE`, `reference_name` matches `REFERENCE_NAME_RE`, `outcome∈OUTCOMES`, **`resolved == (outcome=='resolved')`**, **`manager not in SUPPORTED_MANAGERS ⟹ outcome=='unsupported_manager' and resolved is False`** (B1), provenance rules (caller cannot assert `connector_verified`; connector requires `checked_at`).
- `build_env_outcome(present: bool) -> dict` — `resolved`/`not_found` observation (pure; takes a **boolean**, never a value).
- Pure observation builders for `unsupported_manager` / `probe_error` (honest, `resolved=False`).
